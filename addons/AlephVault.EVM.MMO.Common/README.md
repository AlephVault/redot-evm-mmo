# AlephVault.EVM.MMO.Common

Shared EVM MMO data and helpers used by both the client and server packages.

This package exposes the global namespace `AlephVault__EVM__MMO__Common`.

## SIWE-Like Authentication Helpers

`AlephVault__EVM__MMO__Common.Authentication.Siwe` contains the shared constants
and payload helpers used by the EVM MMO authentication protocols.

The login method is `siwe`. The signed EIP-712 primary type is
`AlephVaultEvmMmoLogin` with this message:

```gdscript
address address
uint32 chainId
string domain
bytes32 nonce
uint32 version
uint64 expirationTime
```

The login payload sent through the base MMO authentication protocol is a
dictionary containing:

- `domain`: game domain/id string.
- `address`: non-zero `0x` EVM address.
- `chainId`: unsigned 32-bit chain id.
- `nonce`: `0x` plus 64 hex digits, generated by the client.
- `version`: integer `1`.
- `expirationTime`: UNIX timestamp in seconds.
- `signature`: `0x` ECDSA secp256k1 signature over the typed data.

Useful helpers:

- `make_payload(domain, address, chain_id, expiration_time, nonce = "")`
- `make_typed_data(payload)`
- `make_typed_data_json(payload)`
- `generate_nonce()`
- `normalize_address(address)`
- `is_valid_address(address)`
- `is_valid_nonce(nonce)`
